# cs330-portfolio

## How do I approach designing software?

This project taught me to break objects down into primitive shapes before writing any code. Looking at a real-world object and planning which primitives could approximate it made the actual coding much more straightforward. I'll carry that decomposition first approach into future projects. 

## How do I approach developing programs?

I build iteratively, adding one object at a time and testing after each addition. When I noticed I was duplicating pencil code, I refactored it into ``RenderPencil`` . 
Over the milestones my code went from flat procedural blocks to organized modular sections. 

## How can computer science help me in reaching my goals?

This course strengthened my understanding of how mathematical concepts like matrix transformations and coordinate systems translate into working code, which reinforces the kind of systematic, low-level thinking that data engineering requires. Graphics may not be my primary path, but understanding how complex systems are built from simple components is a skill that carries into any engineering discipline.
